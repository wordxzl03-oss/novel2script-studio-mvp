# Novel2Script Studio｜AI 小说转剧本工作台

> 面向小说作者与内容创作者的 AI 辅助剧本改编工具：将 3 章以上小说文本转换为结构化 YAML 剧本，并支持 Schema 校验、剧本 Linter、场景级溯源、局部重生成与导出。

## 1. 项目简介

Novel2Script Studio 不是一个简单的“一键小说转剧本”工具，而是一个面向小说改编流程的结构化工作台。

传统 AI 改编工具通常直接输出一段剧本文本，存在三个问题：

1. **不可追溯**：无法判断某一场戏来自原文哪个段落；
2. **不可校验**：人物、地点、场景编号、对白结构容易混乱；
3. **不可局部修改**：用户往往只能重新生成整篇内容，难以对单场戏进行定向调整。

本项目将小说改编过程拆解为：

```text
小说章节
→ 全局角色/地点注册表
→ 场景级 YAML 剧本
→ Schema 校验
→ 剧本 Linter 检查
→ 场景级溯源与局部重生成
→ YAML / Fountain / CSV 导出
```

项目的核心思想是：**将 YAML 作为小说文本与剧本生产之间的中间表示层，使 AI 生成结果可编辑、可校验、可追溯、可继续加工。**

## 2. 核心功能

### 2.1 P0：核心闭环

* 支持上传或粘贴 3 章以上小说文本；
* 自动识别章节结构并完成章节切分；
* 生成全局角色注册表、地点注册表与别名映射；
* 按章节逐步转换为结构化 YAML 剧本；
* 使用 Pydantic 对 YAML 剧本进行 Schema 校验；
* 对非法输出进行自动修复重试；
* 使用剧本 Linter 检查结构性问题；
* 支持场景级原文溯源；
* 支持 YAML 与 Fountain 导出；
* 提供完整的 YAML Schema 设计文档。

### 2.2 P1：增强功能

* 支持选中单个场景进行局部重生成；
* 支持修改前后 diff 对比与确认合并；
* 支持 W/I 级 Linter 规则与一键 AI 修复；
* 支持短剧 profile：自动切集、钩子检测、卡点标记；
* 支持质量指标面板：一次通过率、修复后通过率、实体一致率、情节覆盖率、Lint 密度；
* 支持顺场表 CSV 导出；
* 支持版权确认项与导出署名链。

### 2.3 P2：实验性功能

* AI 围读会：基于角色 id 与 TTS 多音色合成单场音轨；
* 结构仪表盘：张力曲线、人物出场热力图；
* 章节哈希增量缓存：修改后仅重算变化章节；
* 心理描写外化器：将心理描写转化为旁白、动作或对白。

## 4. 工作流

```text
1. 用户上传或粘贴小说文本
2. 系统自动切分章节
3. 系统扫描全局角色、地点、别名与前情摘要
4. LLM 按章节生成场景级 YAML 剧本
5. 后端使用 Pydantic 进行 Schema 校验
6. 若校验失败，系统自动回喂错误信息并修复
7. Linter 检查人物、场景、对白、溯源等结构性问题
8. 前端展示剧本、原文溯源、质量指标与修改建议
9. 用户可局部重生成单场戏
10. 导出 YAML、Fountain 或顺场表 CSV
```

## 5. YAML Schema 设计

本项目的 YAML Schema 不是简单的保存格式，而是小说改编流程中的结构化中间表示。

Schema 主要服务以下目标：

1. **可编辑**：每一场戏、每一句对白、每一个角色都可被定位和修改；
2. **可校验**：通过 Pydantic 检查字段缺失、类型错误、角色未注册等问题；
3. **可追溯**：每个场景都保留来源章节和原文段落区间；
4. **可扩展**：后续可继续导出为 Fountain、分镜脚本、短剧分集表或围读音频；
5. **可评估**：能够统计情节覆盖率、实体一致率、Lint 密度等质量指标。

详细说明见：

```text
docs/schema-design.md
```

## 6. 剧本 Linter 规则

当前 Linter 包含三类规则：

### E 级错误

影响剧本结构有效性的问题，例如：

* E001：场景 id 缺失或重复；
* E002：角色未出现在全局注册表中；
* E003：对白角色未出现在当前场景人物列表中；
* E004：场景缺少来源章节或原文区间；
* E005：剧本中残留明显小说叙述腔。

### W 级警告

影响剧本质量但不阻断导出的问题，例如：

* 场景目标不清晰；
* 冲突不足；
* 对白过长；
* 动作描写不足。

### I 级提示

用于进一步优化表达的问题，例如：

* 可增强镜头感；
* 可压缩旁白；
* 可外化心理描写。

## 7. 质量指标

| 指标      | 含义                      | 采集方式         |
| ------- | ----------------------- | ------------ |
| 一次通过率   | LLM 输出首次通过 Schema 校验的比例 | 流水线埋点        |
| 修复后通过率  | 自愈循环后最终通过 Schema 校验的比例  | 流水线埋点        |
| 实体一致率   | 角色别名归一与跨章人物一致性          | 抽样核对         |
| 情节覆盖率   | 原文关键情节点在剧本中的保留比例        | 情节点抽取 + 人工抽查 |
| Lint 密度 | 每个场景平均错误或警告数量           | Linter 输出    |
| 缓存命中率   | 重转时跳过未修改章节的比例           | 增量缓存统计       |

## 8. 快速启动

### 8.1 后端启动

```bash
cd backend
python -m venv .venv
source .venv/bin/activate  # Windows 可使用 .venv\Scripts\activate
pip install -r requirements.txt
uvicorn app.main:app --reload
```

### 8.2 前端启动

```bash
cd frontend
npm install
npm run dev
```

### 8.3 环境变量配置

复制 `.env.example` 为 `.env`，并填写自己的 API Key：

```bash
cp .env.example .env
```

`.env.example` 中只保留变量名和示例值，不包含真实密钥。

## 11. 开发过程

本项目为 XEngineer 新工科计划第三批议题三「AI 小说转剧本工具」参赛作品。

项目采用 Pull Request 方式持续开发。每个 PR 尽量只完成一个独立功能，并在 PR 描述中说明：

1. 功能描述；
2. 实现思路；
3. 测试方式；
4. 相关截图或运行结果。

## 12. 不做清单

为了保证 72 小时内形成稳定闭环，当前版本暂不实现以下内容：

* 账号体系；
* 富文本长文档编辑器；
* 自训练模型；
* 视频生成；
* 移动端适配；
* 完整商业化部署。
